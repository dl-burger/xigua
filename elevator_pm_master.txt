### 三、团队分工

### 3.1 程序 A — 核心系统

职责：GameManager（全局状态机）、ElevatorController（21层电梯状态机）、SanitySystem（理智值系统）、EventManager（事件触发检测与调度）

交付物：C# 接口定义文档、JSON 配置表规范、核心系统联调测试

### 3.2 程序 B — 事件/UI/输入集成

职责：基于程序A接口搭建事件逻辑框架、实现11个事件的具体表现、输入系统（鼠标点击按钮）、UI框架（理智值HUD/楼层显示）、AudioManager框架

交付物：事件逻辑实现、UI系统、音效集成

### 3.3 美术 A — 电梯/场景/UI

职责：电梯模型（轿厢/门/按钮面板/天花板）、各楼层场景、UI界面（理智值指示器/按钮高亮/暗角特效）

交付物：电梯最终模型、6个楼层场景、UI界面、特效

### 3.4 美术 B — 角色/道具

职责：假模特模型（3个姿态）、NPC（替死人）模型+动画、B1人影、青色手、电梯内道具（灭火器/广告/扶手/合格证）、走廊道具

交付物：角色模型、道具模型、动画

### 3.5 音效师

职责：电梯机械音（5个）、环境音（4个）、UI音效（4个）、事件辅助音效（10个事件配套）、语音台词录制/处理（9段）、玩家呼吸声（4个版本）、事件专属音效（25个）

交付物：约30个音效文件

### 五、项目管理规范

### 5.1 每日站会（15分钟）

每天固定时间，格式如下：

• 昨天完成了什么？

• 今天计划做什么？

• 遇到什么阻碍？

### 5.2 任务看板

推荐使用 TAPD 或腾讯文档在线表格，列名：

• 📋 TODO  • 🔄 进行中  • 👀 待测试  • ✅ 已完成  • ❌ 已阻塞

每个任务需标注：负责人 / 预计工时 / 截止日期

### 5.3 Git 分支规范

• main 分支：正式版本，仅限 Phase 里程碑合并

• dev 分支：日常开发集成

• feature/xxx 分支：各人功能分支

• 提交规范：[类型] 描述，如 [Event] 实现事件04人偶入电梯逻辑

• Unity 场景管理：禁止多人同时修改同一场景，使用 Prefab 拆分

### 5.4 文件命名规范

美术文件：ELEV_轿厢.fbx / SCENE_B1 / UI_理智值.png / FX_灯光闪烁.shader

音效文件：SFX_mech_elevator_start.wav / SFX_ambient_corridor_8F.wav / VO_001.wav
